# Publishing CechCore_jll and TopoTS.jl

This document describes the exact steps to go from the current state
(working local build, tested ABI) to:

1. `CechCore_jll` in the Julia General registry (pre-built binaries for all platforms)
2. `TopoTS` in the Julia General registry (`]add TopoTS` just works)

---

## Part 1 — Publish CechCore_jll

### Step 1: Create the GitHub repository

```bash
gh repo create shalkiewicz/CechCore_jll.jl --public
```

The repo needs exactly this layout (BinaryBuilder convention):

```
CechCore_jll.jl/
├── Project.toml      ← from jll/CechCore_jll/Project.toml
├── Artifacts.toml    ← generated by BinaryBuilder (replaces our stub)
└── src/
    └── CechCore_jll.jl   ← from jll/CechCore_jll/src/CechCore_jll.jl
```

### Step 2: Tag a source release of TopoTS

BinaryBuilder needs a stable source URL. Create a git tag:

```bash
cd <TopoTS_root>
git tag v0.3.0
git push origin v0.3.0
```

Update `jll/build_recipe/build_tarballs.jl` line ~30:
```julia
GitSource("https://github.com/shalkiewicz/TopoTS.jl.git", "v0.3.0")
```

### Step 3: Run BinaryBuilder

Install BinaryBuilder once:
```julia
]add BinaryBuilder
```

Run the build (takes ~20 minutes; cross-compiles for all platforms in Docker):
```bash
julia jll/build_recipe/build_tarballs.jl --verbose
```

For a faster first test, build one platform only:
```bash
julia jll/build_recipe/build_tarballs.jl --verbose x86_64-linux-gnu
```

BinaryBuilder produces:
- `products/CechCore.v1.0.0.*.tar.gz`  — one tarball per platform
- `Artifacts.toml`  — with real sha256 and git-tree-sha1 hashes

### Step 4: Publish tarballs as GitHub releases

```bash
# Create a GitHub release and upload all tarballs
gh release create CechCore-v1.0.0+0 products/*.tar.gz \
    --repo shalkiewicz/CechCore_jll.jl \
    --title "CechCore v1.0.0+0" \
    --notes "Pre-built libcech binaries for all supported platforms."
```

### Step 5: Finalise the JLL package

Copy the BinaryBuilder-generated `Artifacts.toml` into the JLL repo:
```bash
cp Artifacts.toml path/to/CechCore_jll.jl/Artifacts.toml
```

Commit and tag:
```bash
cd CechCore_jll.jl
git add .
git commit -m "CechCore v1.0.0+0 — all platforms"
git tag v1.0.0+0
git push origin main v1.0.0+0
```

### Step 6: Register CechCore_jll in General

```julia
using Registrator
@register  # run from inside the CechCore_jll.jl directory
```

Or use the Registrator GitHub App: open a PR from
https://github.com/JuliaRegistries/Registrator.jl

After the PR is merged (~3 days for review), users can:
```julia
]add CechCore_jll
```

## Part 2 — Publish via Yggdrasil (preferred for General packages)

For packages whose binaries belong in the official Yggdrasil tree
(better long-term maintenance, automatic rebuilds):

```bash
# Fork Yggdrasil
gh repo fork JuliaPackaging/Yggdrasil --clone

# Add the build recipe (rename to match Yggdrasil convention)
mkdir -p Yggdrasil/C/CechCore
cp jll/build_recipe/build_tarballs.jl Yggdrasil/C/CechCore/build_tarballs.jl

# Open a PR — Yggdrasil CI builds all platforms and auto-creates the JLL repo
gh pr create --repo JuliaPackaging/Yggdrasil \
    --title "Add CechCore" \
    --body "Čech filtration C++ library for TopoTS.jl TDA package."
```

Yggdrasil CI handles all platform builds, uploads the tarballs, creates
`CechCore_jll.jl`, and submits it to the General registry automatically.

## Part 3 — Register TopoTS in General

Once `CechCore_jll` is registered:

1. Update `TopoTS/Project.toml` `[weakdeps]` UUID to match the registered JLL
2. Tag `TopoTS v0.3.0`
3. Register via Registrator:
   ```julia
   using Registrator
   @register
   ```

After registration, the full zero-compilation workflow works:
```julia
]add TopoTS
]add CechCore_jll          # optional — enables :cech filtration

using TopoTS
dgms = persistent_homology(emb; filtration=:cech)   # just works
```

## Part 4 — Local development workflow (no GitHub needed)

While the above is in progress, the local build workflow continues to work:

```bash
cd csrc && make             # builds deps/lib/libcech.so
```

```julia
using TopoTS
cech_available()            # → true  (local build found)
persistent_homology(emb; filtration=:cech)
```

You can also point to any libcech build without recompiling:
```julia
ENV["TOPOTS_LIBCECH"] = "/path/to/libcech.so"
using TopoTS
```

## Verification checklist

Before submitting to either registry, verify:

- [ ] All 11 C++ smoke tests pass: `cd csrc && make test`
- [ ] ABI struct size == 48: confirmed by test
- [ ] `cech_available()` returns `true` after `]add CechCore_jll`
- [ ] `persistent_homology(emb; filtration=:cech)` runs on all five platforms
- [ ] Čech/Rips interleaving test passes (see `test/runtests.jl`)
- [ ] `]test TopoTS` passes with CechCore_jll installed
- [ ] `]test TopoTS` passes *without* CechCore_jll (graceful degradation)
